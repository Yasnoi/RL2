## Rollout 代码研读总结

### 核心架构
- **Rollout 类**：负责任务编排和推理服务器管理
- **SampleGroup**：一个 prompt 的多个 response（通过 responses_per_prompt 控制）
- **Sample**：单个 trajectory，包含 state、action、reward 序列

### 关键流程
1. 初始化：启动 SGLang 推理服务器 + Router
2. 推理循环（`__call__`）：
   - rank 0 调度 asyncio 任务并发生成
   - 每个 Sample：LLM 生成 → env.step() → 获取 reward → 判断是否结束
3. 权重更新（`update`）：训练后将权重同步到推理服务器

### Train-Inference Mismatch 注入点（Rollout）

1. **LLM 生成参数扰动**：sampling_params (temperature, top_p 等)
2. **环境 step 扰动**：env_step 函数中的 reward/next_state
3. **Response 后处理扰动**：add_llm_response 函数
4. **Rollout 层整体扰动**：sample_group 结果

## PPO 代码研读总结

### 8 步训练主循环
1. Rollout：生成训练数据
2. Reference Logps：计算参考策略 logps（如果启用 KL）
3. Values：计算状态值（如果使用 GAE）
4. Actor Logps：重新计算 actor logps
5. Advantages：计算优势函数（REINFORCE 或 GAE）
6. PPO Update：更新 actor/critic
7. Checkpoint：保存检查点
8. Rollout Update：同步权重到推理服务器

### 优势估计方法
- **REINFORCE**：简单高效，适合稀疏 reward
- **GAE**：需要 Critic，通过 γ 和 λ 平衡偏差和方差

### Train-Inference Mismatch 注入点（PPO）

1. **Rollout 生成参数**：影响 action 分布
2. **Reward 信号扰动**：影响 advantage 计算
3. **Advantage 计算扰动**：影响策略梯度方向
4. **权重同步扰动**：推理时模型与训练不一致
